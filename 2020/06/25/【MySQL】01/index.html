【MySQL】01

数据库的介绍

保存数据的容器：

  • 数组 内存
  • 集合 内存
  • 文件 不太适合查询

    DB（database）：存储数据的仓库

    DBMS（Database Management System) ：

  • 数据库管理系统
  • 常见的数据库管理系统：MySQL、Oracle、DB2、sqlserver

    SQL(Structure Query Language):

  • 结构化查询语言
  • 专门与DBMS通信的语言

    特点

  • 数据->表->库
  • 库里可以有多个表，每个表都有一个名字，表名具有唯一性
  • 表：特性，这些特性定义了数据在表中如何存储
  • 表由列组成 列：字段
  • 表按行存储

MySQL的常见命令

  • 查看当前所有的数据库:
    show databases;
  • 打开指定的库:
    use 库名;
  • 查看当前库的所有表:
    show tables;
  • 查看其他库的所有表:
    show tables from 库名;
  • 创建表:
  • 查看表结构:
    desc 表名;
  • 查看服务器的版本:
    方式一：登录到mysql服务端
    select version();
    方式二：没有登录到mysql服务端
    mysql --version或mysql --V

MySQL的语法规范

  • 不区分大小写，但建议关键字大写，表名、列名小写
  • 每条命令最好用分号结尾，也可以用\g
  • 每条命令根据需要，可以进行缩进或换行
  • 注释
    单行注释 ：#注释文章
    单行注释 ：- - 注释文字（有空格）
    多行注释 ：/* 注释文字 */

DQL语言（查询）

基础查询

语法：
select 查询列表 from 表名;

  • 可以查表中的字段、常量值、表达式、函数
  • 查询的结果是一个虚拟的表格:

例如
查询表中的单个字段:
select last_name from employees;
查询表中的多个字段:
select last_name,salary,email from employees;
(用逗号隔开)
查询表中的所有字段:
select * from employees;
(顺序就不能更改了)
着重号`name`可以用来区分是关键字还是字段
查询常量值:
select 100; select 'join';
(数据库中没有字符串概念)
查询表达式:
select 100*98;
查询函数:
select version();

  • 起别名(①便于理解②可以区分重名)

方式一 使用as
select 100*98 as 结果;
select last_name as 姓,first_name as 名 from employees;
方式二 使用空格
select last_name 姓 from employees;
如果别名里有空格，出现歧义，加双引号

  • 去重
    select distinct department_id from employees;
  • +号的作用

+号：运算符 转换失败为0，有一方为Null, 结果为Null
连接的话 用CONCAT;
select CONCAT(last_name,first_name) as 姓名 from employees;

  • 如果拼接的时候 里面有Null 用函数判断 select
    *
    from
    employees
    where
    last_name LIKE ‘%a%’; select
    *
    from
    employees
    where
    last_name LIKE ‘__e_a%’; select
    *
    from
    employees
    where
    last_name LIKE ‘\%’;
    或者 last_name LIKE ‘$%’ escape ‘$’; select
    *
    from
    employees
    where
    employee_id between 100 and 120; select
    last_name,
    job_id
    from
    employees
    where
    job_id = ‘IT_PROT’ or job_id=’AD_VP’ or job_id=’AD_PRES’;
    也可以
    job_id IN (‘IT_PROT’,’AD_VP’,’AD_PRES’); select
    last_name,
    commission_pct
    from
    employees
    where
    commission_pct is NULL;
